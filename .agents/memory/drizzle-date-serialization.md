---
name: Drizzle date serialization
description: Drizzle ORM returns Date objects but Orval-generated Zod response schemas use zod.string() for date fields. Must serialize before calling .parse().
---

Drizzle ORM returns `Date` objects for `timestamp` columns, but the Zod response schemas generated by Orval (from OpenAPI `format: date-time` fields) use `zod.string()`. Calling `.parse()` directly on Drizzle results throws a ZodError.

**Fix:** Use `serializeDates` / `serializeList` helpers in `artifacts/api-server/src/lib/serialize.ts` before any `.parse()` call.

```ts
import { serializeList, serializeDates } from "../lib/serialize";
res.json(ListTransactionsResponse.parse(serializeList(txns)));
res.json(GetTransactionResponse.parse(serializeDates(txn)));
```

**Why:** OpenAPI spec uses `format: date-time` → Orval generates `zod.string()` → Drizzle returns `Date` → mismatch at runtime, 500 errors on every list/get endpoint.

**How to apply:** Any new route that returns DB rows with timestamp columns must go through `serializeDates/serializeList` before Zod parsing.
